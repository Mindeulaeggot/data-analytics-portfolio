# Portfolio Review Rubric

I will review each project against these questions before calling it portfolio-ready.

## 1. Clarity

- Can a recruiter understand the project in under 30 seconds?
- Does the README open with a business problem, not just tools?
- Are the results easy to find?

## 2. Evidence

- Are there concrete metrics?
- Are there charts, outputs, or tables?
- Is the workflow reproducible?

## 3. Relevance

- Does the project match data analyst or analytics intern work?
- Does it include business interpretation, not only code?
- Does it show practical decision-making value?

## 4. Technical Quality

- Is the code readable?
- Are preprocessing and evaluation choices explained?
- Are file names and folder structures clean?

## 5. Presentation

- Is the GitHub README polished?
- Are the best repositories pinned?
- Is the strongest project placed first in the portfolio?

## 6. Upgrade Trigger

If a project lacks any of these, it needs another revision:

- clear business framing
- meaningful evaluation metrics
- visual evidence
- concise summary bullets for a resume or interview
